# 기본 작업 규칙

- 수정 전에는 먼저 변경 계획을 요약한다.
- 파일 수정 후에는 변경된 파일 목록과 핵심 diff를 요약한다.
- HTML, CSS, JavaScript를 수정한 경우 문법 검사만으로 완료 처리하지 않는다.
- 실제 브라우저 검수를 반드시 시도한다.
- 브라우저 검수가 실패하면 커밋/푸시하지 말고 원인과 수정안을 먼저 보고한다.
- 커밋/푸시는 사용자가 명시적으로 승인한 경우에만 진행한다.
- 배포 후에는 GitHub Actions 또는 배포 로그에서 성공 여부를 확인한다.

# 브라우저 검수 필수 항목

- 데스크톱 화면에서 페이지가 정상 렌더링되는지 확인한다.
- 모바일 390px 화면에서 레이아웃이 깨지지 않는지 확인한다.
- 주요 버튼을 실제로 클릭해 동작을 확인한다.
- 콘솔 에러가 없는지 확인한다.
- 새로고침 후 저장 데이터가 유지되는지 확인한다.
- 삭제/초기화 기능이 있으면 실제로 동작하는지 확인한다.

# 로또 번호 생성기 검수 항목

- 1게임 생성 버튼 클릭 시 번호 6개가 나온다.
- 5게임 생성 버튼 클릭 시 5게임이 나온다.
- 각 게임의 번호는 1~45 범위다.
- 각 게임 안에서 번호 중복이 없다.
- 복사 버튼이 정상 동작하거나, 브라우저 제한 시 안내 문구가 나온다.
- 생성 히스토리가 추가된다.
- 새로고침 후 히스토리가 유지된다.
- 전체 삭제 버튼 클릭 시 히스토리가 삭제된다.
- 모바일 화면에서 공, 버튼, 히스토리 영역이 화면 밖으로 넘치지 않는다.

# 완료 보고 형식

작업 완료 보고에는 반드시 아래를 포함한다.

- 수정한 파일
- 핵심 변경사항
- 실행한 검증 명령어
- 브라우저 검수 결과
- 실패하거나 못 한 검수 항목
- 커밋/푸시 여부
- 배포 성공 여부
